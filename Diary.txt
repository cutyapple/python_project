체크:
    체크의 경우, 현재 두 가지의 체크가 존재한다.
    1. 상대방이 움직여 내 킹이 체크되는 경우. (완료)
    2. 나의 말이 움직여 내 킹이 체크되는 경우. (고민중)
        2번의 경우는 체스 룰 상 허용되지 않는 움직임이다. 로직에 대해 깊은 탐구가 필요 해 보인다.

    [B의 턴시작] -> <B의 왕이 체크 상태인가?>
        (True)->  [B의 말을 선택] -> [말의 이동 가능 범위 파악], [체크한 말의 공격 범위 파악]
    { 이때, 나이트나 폰같은 특수경우도 예외처리로 고려해야 함 } -> [적팀의 모든 말의 공격 범위 파악] {이때의 공격 범위는, 선택한 말의 위치를 무시하고 지정함}
    -> [이중 for문으로 두 개의 리스트를 비교하여, 만일 적팀의 공격 범위에 B팀의 왕의 location에 닿는 다면 이동 불가 지정]
        (False)->  [B의 말을 선택] -> [말의 이동 가능 범위 파악], [체크한 말의 공격 범위 파악]
    { 이때, 나이트나 폰같은 특수경우도 예외처리로 고려해야 함 } -> [적팀의 모든 말의 공격 범위 파악] {이때의 공격 범위는, 선택한 말의 위치를 무시하고 지정함}
    -> [이중 for문으로 두 개의 리스트를 비교하여, 만일 적팀의 공격 범위에 B팀의 왕의 location에 닿는 다면 이동 불가 지정]

    즉, 체크의 유무는 코드에 영향을 주지 않는 비가변적 요소이다.

        for m_name, m_list in piece_move_list:  #말의 이동 가능 범위 list에서 각 요소들(m_name, m_list)을 빼옴
